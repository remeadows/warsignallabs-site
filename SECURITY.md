# Security Policy

## Supported Versions

Security fixes are targeted at:

- The latest commit on the default branch
- The most recent tagged release, if this repository publishes releases

Older versions, stale branches, snapshots, and archived repositories may not receive security updates.

## Reporting a Vulnerability

Please report security issues privately to `rmeadows@warsignallabs.net`.

Do not open public GitHub issues, discussions, or pull requests for vulnerabilities that could expose users, systems, credentials, or operational details.

Include the following where possible:

- Repository name and affected branch, tag, or commit
- Affected files, endpoints, workflows, or components
- Impact and required preconditions
- Reproduction steps or a proof of concept
- Relevant logs, screenshots, stack traces, or indicators of exposure

## Response Expectations

- Acknowledgement target: within 5 business days
- Triage and validation updates will follow as the issue is reproduced and scoped
- If the report is confirmed, remediation and disclosure timing will be coordinated privately

## Disclosure Guidelines

- Please allow a reasonable remediation window before public disclosure
- Do not access, modify, or exfiltrate data beyond what is necessary to demonstrate the issue
- If the issue involves exposed credentials or tokens, stop using them and report the exposure details immediately

## Scope

- This policy applies to this repository only
- Archived branches, unsupported versions, and historical snapshots are out of scope for guaranteed fixes
- This project does not currently offer a public bug bounty program
