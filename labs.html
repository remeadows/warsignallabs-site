Research

A research line, and a portfolio that pays its tuition.

WarSignalLabs runs an active research line on multi-agent AI communication. It also ships iOS games — not as the business, but as an honest pressure-test for the production discipline the consulting work depends on.

Research direction

Toward formal models of agent communication.

Multi-agent AI systems work by passing messages between agents. Today those protocols are largely ad-hoc: handcrafted prompts, JSON schemas, and convention. They work well enough for demos, badly enough that nobody is comfortable putting them inside systems where a wrong answer has consequences.

WarSignalLabs is building a repeatable local-AI research pipeline to study what changes when an agent’s behavior is shaped by structured corrections rather than left to the base model alone. The goal is not impressive answers. The goal is measurable progress — or measurable failure — with evidence.

The method

Take a base language model. Measure how it behaves on a fixed task before any intervention. Then layer in two kinds of correction — a protocol or prompt correction, and a fine-tuning correction via LoRA or QLoRA — and measure again. The same pipeline runs every time. The same scoring runs every time. What changes is what the corrections do.

The behavior of an agent at any stage decomposes additively:

# Components
f₀(x)  = base model behavior
Δf₁(x)  = protocol / prompt correction
Δf₂(x)  = LoRA correction
# Composed behavior
f₂(x)  = f₀(x) + Δf₁(x) + Δf₂(x)
# Equivalently, in named form
ffinal(x) ≈ fbase(x) + Δfprotocol(x) + ΔfLoRA(x)

The model is simple by design: each correction is a measurable contribution that can be attributed, isolated, and rolled back. An agent that produces a bad answer can be queried about which correction caused it. An agent that improves can be queried about which correction earned the improvement. That attribution is the property that turns a black-box workflow into a system a regulated buyer can sign off on.

Why this matters

Reliability

Cybersecurity and scientific reasoning need agents that behave predictably under adversarial input. Additive, attributable corrections make behavior auditable instead of mysterious.

Explainability

If an agent gets something wrong, the decomposition tells you which layer is responsible. Regulated buyers will not adopt agents they cannot debug.

Local first

The whole pipeline is designed to run on local hardware. No cloud dependency, no data leaving the network, no external API contract to break.

Applied work

GW-OS — Edge AI in the field.

The research above informs an active applied project. GW-OS is a small fleet of Raspberry Pi devices running local AI models that watch a network and report what they see. It is the practical end of the question: how do small AI agents coordinate, in production, with bounded behavior?
